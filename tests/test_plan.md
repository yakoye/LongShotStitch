# LongShotStitch v1.47.0 Test Plan

## v1.47.0 移动标注与统一属性回归

1. 单行文字在蓝色选中框内上下视觉居中；Canvas、选择框和 textarea 的外框尺寸一致。
2. 手机端新建文字使用贴合对象的输入框，不得变成占满屏幕或底部的大输入面板。
3. 手机端单击已有文字只选择和拖动；再次轻点打开属性；双击或点击编辑按钮才进入文字内容编辑。
4. 手机端选择矩形、画笔、箭头、标号、马赛克、水印、贴图和浮动选区时，第一次轻点均不得自动弹出遮挡对象的属性。
5. 用户主动打开属性后，对象仍保持可见；拖动、缩放或旋转开始时属性自动收起。
6. 小文字、小标号等对象中心可以拖动，不被四角控制点完全覆盖。
7. 手机端和桌面端工具顺序均为：选择、选区、形状、画笔、箭头、文字、标号、马赛克、橡皮、水印。
8. 桌面端工具呈 5×2 网格，属性面板不超过 300px 侧栏，不覆盖画布。
9. 所有属性按钮、数字输入、文本输入和颜色按钮保持统一紧凑尺寸；图层与对象操作在一行内。
10. 390×844 手机视口逐项打开全部十组工具属性，无横向页面溢出、无底部遮挡。
11. 使用真实 Chromium 创建形状、铅笔、箭头、文字、标号、马赛克、水印、浮动选区并执行局部橡皮。
12. 运行三个浏览器专项测试和全部 Node 回归，均必须通过。

## v1.46.0 标注选择与紧凑导出回归

1. 选择铅笔或荧光笔，按下并拖动画线；拖动过程中不得出现蓝色虚线框、缩放点或旋转点。
2. 松开鼠标后，刚完成的轨迹保持选中，显示起点、终点、缩放与旋转控制，并继续保持铅笔/荧光笔工具可直接绘制下一笔。
3. 依次选择矩形、圆形、直线、箭头、铅笔、荧光笔、文字、标号、马赛克、水印和粘贴图片；每次单击对象后都必须稳定显示该对象对应的属性，不得随机隐藏或显示上一工具属性。
4. 调整图层后继续修改当前对象属性，确认选择通过对象 ID 保持，不会跳到相邻对象。
5. 在 1366×768 和 1920×1080 桌面视口打开导出菜单，面板宽度不得超过 360 px，且宽、高、毫米和 DPI 输入框只保留数字所需宽度。
6. 在 390 px 手机视口打开导出菜单，面板不得超出屏幕，格式、尺寸、预设和质量控件可完整操作。

- 电脑端模式按钮、图片列表、属性卡片更紧凑，侧栏信息密度明显提高。
- 电脑端裁切属性只占两行：分割一行、裁剪一行；行/列、比例、自定义和 ok 在同一行内完成。
- 电脑端整体裁剪、单图裁剪的上下左右输入为两行紧凑布局，不再四行铺满。


## v1.31 manual checks

- 分割线/拼接线编辑态拖动上方或左方图片时，分割线和下方/右方图片在整个拖动过程中保持屏幕位置不动。
- 分割线/拼接线编辑态拖动下方或右方图片时，分割线和上方/左方图片在整个拖动过程中保持屏幕位置不动。
- 图片较小或完全落在可视区域内时，拖动中不应触发自动居中，也不应出现两边同时收紧。
- 只有点击对号退出编辑态后，画面才允许重新居中或回正。


## v1.32 manual checks

- 横向拼接线编辑态拖动左图向右时，拼接线和右图保持屏幕位置不动，只有左图内容移动。
- 横向拼接线编辑态拖动右图向左时，拼接线和左图保持屏幕位置不动，只有右图内容移动。
- 整体裁剪、单图裁剪进入上下左右编辑态后，拖动过程中不应被自动适合窗口/自动居中抢走位移。
- 拖动停止但未点击对号时，画面仍保持编辑中的位置；点击对号后才允许重新居中或回正。


## v1.33 manual checks

- 整体裁剪调整后点击对号，裁剪结果应固化为新的工作底图，图片列表尺寸同步变小。
- 整体裁剪已有裁剪值时，直接切到单图裁剪/拼接线/分割线/裁切，应先固化再进入新工具。
- 固化后整体裁剪值归零，单图裁剪的上下左右输入从 0 开始，后续裁切导出基于裁剪后的图片。
- 如果整体裁剪没有裁掉任何边，点击对号或切换工具不应生成新的图片记录。


## v1.34 manual checks

- 整体裁剪完成后不应合成单图；多张图结构和可用拼接线应保留。
- 纵向整体裁剪裁掉顶部/底部整张图时，被完全裁掉的图片删除，剩余图片仍可单图裁剪和拼接线调整。
- 横向整体裁剪裁掉左侧/右侧整张图时，被完全裁掉的图片删除，剩余图片仍可单图裁剪和拼接线调整。
- 手机端拼接线按钮带 `···` 提示；弹出的四个拼接功能按钮一行等宽，隐藏菜单后当前字幕/拼接功能仍可操作。
- 电脑端没有独立字幕模式按钮；拼接线属性中显示四个等宽按钮：手动拼接、自动拼接、自动字幕、手动字幕。
- 单图裁剪图标为方框内数字 1，手机端和电脑端一致。


## v1.35 manual checks

- 先调整拼接线，再做竖向整体裁剪的左右边，完成后图片数量不应减少，高度不应被重复裁短。
- 先调整拼接线，再做横向整体裁剪的上下边，完成后图片数量不应减少，宽度不应被重复裁短。
- 整体裁剪完成后保留当前缩放比例，不自动执行适合窗口，不出现突然放大或缩小。


## v1.36 manual checks

- 手机端裁切中，从“分割”切到“裁剪”或从“裁剪”切到“分割”时，只切换功能，不立即弹出二级菜单。
- 手机端裁切中，当前已是“分割”时再次点击“分割”才弹出/隐藏分割参数；“裁剪”同理。
- 手机端拼接中，从其它拼接功能切回“拼接线”时只切功能，再次点击“拼接线”才弹出/隐藏手动拼接、自动拼接、自动字幕、手动字幕。


## v1.37 manual checks

- 分割线在竖向拼接的大图中插入后，进入编辑态的接缝应停在原插入线位置，不向上跳。
- 分割线在横向拼接的大图中插入后，进入编辑态的接缝应停在原插入线位置，不向左跳。
- 低缩放下拖动分割线插入线应能直接命中，不需要等待、放大缩小或反复点击。
- 拼接线/分割线编辑态拖动上方、下方、左方、右方图片时，手感应一致；大图被缩放后也不应明显变慢。
- 整体裁剪完成后不显示“已分摊锁定”等内部实现提示。


## v1.38 manual checks

- 空状态虚线区域点击空白处不应弹出打开图片或打开工程，只让虚线区域获得焦点。
- 空状态虚线区域获得焦点后，Ctrl+V / 粘贴事件仍可导入剪贴板图片。
- 空状态只在点击“打开图片”“打开工程”“粘贴”三个按钮时执行对应操作。
- 空状态“粘贴”按钮位于“打开工程”右边。


## v1.39 manual checks

- 手机端点击顶部 `i` 按钮应弹出信息面板，而不是无效按钮。
- 信息面板应显示版本、当前模式、方向、图片数、输出尺寸、缩放和预览状态。
- 空项目时点击 `i`，信息面板应显示版本和等待导入状态。


## v1.40 manual checks

- 电脑端整体裁剪或放大图片后，上下左右滚动到边缘仍应保留安全边距，底部/侧边裁剪手柄和按钮不贴屏幕边缘。
- 电脑端顶栏文件入口应归为一个“文件”按钮，菜单内包含打开图片、打开工程、粘贴、保存工程和关于版本。
- 手机端文件菜单仍保持简洁，不显示桌面专用的粘贴入口。
- 手机端和电脑端点击 `i` 都应显示半透明信息面板，再点一次关闭；底部状态条不应显示。
- 导出格式和手机端裁剪比例不应触发安卓/系统全屏原生下拉，应在当前按钮附近显示自绘选项。
