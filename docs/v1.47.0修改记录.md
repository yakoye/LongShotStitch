# v1.47.0 修改记录

日期：2026-07-17

## 版本与开发约束

- 应用版本升级为 `v1.47.0`。
- 将 `POWERSTORM_REQUIREMENTS.md` 正式改名并重构为 `SUPERPOWERS_REQUIREMENTS.md`。
- 新增长期专项规则 `docs/ANNOTATION_UI_CHECKLIST.md`，统一桌面端、手机端、文字、属性浮层和全工具验证标准。
- HTML 入口顶部改为引用 `SUPERPOWERS_REQUIREMENTS.md`。

## 文字布局与编辑

- Canvas、选中框和行内编辑器共用同一套字号、行高、内边距及实际字体 ascent/descent 测量。
- 文字改用 alphabetic baseline 和计算后的 `baselineOffset`，单行文字在选中框中上下视觉居中。
- 行内编辑器统一使用 `border-box`，移除手机端额外的 44px 最小宽度，外框不再大于真实文字对象。
- 手机端移除固定大输入面板，文字编辑框直接贴合对象。
- 已有文字单击只选择；双击或属性中的编辑按钮才进入内容编辑。
- 桌面端延迟 30ms 获取文字输入焦点，避免 Canvas 指针松开后立即使编辑器失焦关闭。

## 手机端选择与属性

- 第一次轻点对象只选择、移动、缩放或旋转，不自动弹出属性遮挡对象。
- 再次轻点已经选中的对象，才切换该对象的属性浮层。
- 拖动、缩放和旋转开始时主动收起手机属性。
- 用户主动打开属性后执行遮挡检测，必要时轻微移动画布，使对象保持可见。
- 小文字、小标号等对象增加中心拖动命中区，避免四角控制点覆盖整个对象导致无法移动。

## 工具顺序与统一控件

- 手机端工具顺序与桌面端统一为：选择、选区、形状、画笔、箭头、文字、标号、马赛克、橡皮、水印。
- 桌面端标注工具改为稳定的 5×2 网格，移除会占据额外布局位置的分隔元素。
- 常规属性按钮统一为 28px 高；数字输入框、文字输入框、颜色按钮和弹层圆角统一。
- 图层与复制、删除、收起合并为紧凑单行；图层标签收紧为“层”。
- 手机端隐藏不适合触摸的 X/Y/宽高/旋转数字输入，仍可通过画布控制点操作。
- 桌面端属性浮层宽度限制在左侧栏内，属性行内部滚动，不再覆盖画布。

## 全工具检查

- 实际 Chromium 中逐项创建并验证：形状、铅笔、箭头、文字、标号、马赛克、水印、选区浮动贴片和局部橡皮。
- 手机端逐项打开十组工具属性，检查边界、高度、横向溢出和触摸按钮尺寸。
- 验证新建对象、重新选择、对应属性、复制、删除和浮层收起入口。
- 保留既有 Shift 约束、图层、连续绘制、局部橡皮、扩图、删除收缩和指定像素导出能力。
