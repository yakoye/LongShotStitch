# v1.46.0 修改记录

## 版本

- 应用版本升级为 `v1.46.0`。
- `LongShotStitch.html` 与发布入口 `index.html` 保持完全一致。

## 连续画笔显示

- 新建铅笔、荧光笔和自由马赛克时，草稿对象不再写入当前选择状态。
- `drawToolsOverlay()` 明确识别正在创建的 `toolCreate` 草稿；即使选择状态发生变化，也不绘制草稿的虚线框和控制柄。
- 有效笔画提交后保持选中，显示自由路径首尾标记、8 点缩放框和旋转控制。
- 提交后仍保持原连续工具激活，可以直接开始下一笔。

## 属性面板稳定性

- 新增 `selectedAnnotationIndex()`、`selectedAnnotation()` 与 `setSelectedAnnotation()`。
- 选择记录保存对象 ID，并在读取时按 ID 重新解析当前索引，避免图层重排后索引失效。
- 画布选择、控制柄、文字双击、新建、粘贴、复制、图层操作、剪贴板和键盘移动统一使用稳定选择入口。
- `selectAnnotationForEdit()` 在桌面端和移动端都会明确打开对象属性面板。
- 水印等特殊入口也使用同一选择函数，不再构造缺少 ID 的临时选择对象。

## 导出界面

- 导出浮层桌面端宽度限制为最大 `360px`，移动端限制在视口内。
- 输出宽、高输入框收窄到约 `76px`。
- 毫米宽、高输入框收窄到约 `72px`，DPI 输入框约 `82px`。
- 毫米/DPI 行改为固定紧凑列，避免原生数字输入框把整个浮层撑宽。
- 数字输入使用右对齐和等宽数字，尺寸关系更容易快速查看。

## 验证

- 新增 `tests/annotation_selection_export_check.js`。
- 静态回归覆盖草稿虚线框抑制、自由路径首尾标记、稳定选择函数、属性面板打开以及紧凑导出 CSS。
- Chromium DevTools Protocol 真实运行验证覆盖：按住绘制期间无虚线调用、松开后对象属性可见、取消后重新选择属性仍可见、导出面板及数字输入框实际渲染宽度。
