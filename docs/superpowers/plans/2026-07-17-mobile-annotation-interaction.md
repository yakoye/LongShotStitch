# LongShotStitch v1.47.0 实施计划

## 阶段 1：约束与测试

1. 将 `POWERSTORM_REQUIREMENTS.md` 重构并改名为 `SUPERPOWERS_REQUIREMENTS.md`。
2. 添加 v1.47.0 手机端标注专项静态测试。
3. 添加 Chromium 手机端真实交互测试。
4. 先运行测试并确认新测试在旧实现上失败。

## 阶段 2：文字布局

1. 统一 Canvas 和 textarea 的字体测量。
2. 使用 ascent/descent 计算文字基线。
3. 收紧文字选择框内边距。
4. 移除手机端全屏式文字输入框。
5. 保持新建文字立即编辑；已有文字双击或显式操作编辑。

## 阶段 3：选择与属性

1. 手机端单击对象只选择和变换，不自动打开属性。
2. 再次轻点已选对象切换属性浮层。
3. 拖动、缩放、旋转开始时关闭手机属性浮层。
4. 属性主动展开时执行遮挡检测和画布让位。
5. 保持桌面端侧栏属性即时显示。

## 阶段 4：控件和工具顺序

1. 手机端工具顺序与桌面端统一。
2. 统一按钮、输入框、颜色、数字和弹层样式。
3. 图层与删除/复制/收起合并为紧凑单行。
4. 手机端隐藏坐标类精确输入，减少无效占屏。
5. 检查全部十组标注工具的属性布局。

## 阶段 5：回归与发布

1. 更新版本号、README、修改记录和测试计划。
2. 同步 `LongShotStitch.html` 与 `index.html`。
3. 执行全部 Node 测试、JS 语法检查、diff 检查。
4. 使用 Chromium 分别验证 390px 手机端和桌面端。
5. 打包 `LongShotStitch-v1.47.0.zip` 并解压后二次验证。
