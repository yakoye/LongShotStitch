# 标注选区与可编辑对象设计

## 目标

在现有单文件 Canvas 应用上扩展统一对象模型，使矩形、椭圆、自由选区能够自动变成可移动缩放的图像对象；同时让系统剪贴板图片在标注模式下粘贴为可再次编辑的对象，并把左侧“工具”入口统一改为“标注”。

## 方案选择

### 方案 A：临时选区框继续等待复制/剪切

改动最小，但仍然不能满足“选中后自动转成变换工具”，用户还要多执行一次复制或剪切再粘贴。

### 方案 B：选区完成后生成浮动 patch，并保留底图原内容

实现简单，但移动后会看到源位置和浮动对象各有一份，行为更像复制，不像 Windows 画图的选区。

### 方案 C：选区完成后生成 patch，同时生成同形状 cutout（采用）

选区留在原位时画面不变；移动或缩放后，源位置被透明挖空，接近画图的浮动选区。patch 和 cutout 都是非破坏对象，可撤销、保存和再次编辑。

## 状态模型

- `toolSettings.selectionShape`: `rect | ellipse | free`。
- `selectionDraft`: 临时选区，包含 `shape`、包围框和自由路径点。
- `patch`: 浮动图像对象，保存透明 PNG、位置、尺寸和旋转。
- `cutout`: 底图透明挖空对象，保存同形状和相对路径。
- `selectionBusy`: 防止异步提取期间重复触发。
- 完成提取后统一调用 `switchToTransformTool(index)`，把工具组设为 `select`、选中新对象并重绘属性面板。

## 路径与像素提取

- 矩形：直接按包围框裁剪。
- 椭圆：在离屏 Canvas 中按椭圆路径裁剪，外部保持透明。
- 自由：记录世界坐标点，闭合成多边形，按路径裁剪。
- 所有形状都先限制在当前合成画布范围内，并受最大边长、最大像素数保护。
- cutout 使用与提取相同的相对路径，确保源位置挖空边界一致。

## 粘贴规则

- 有底图且当前处于 `tools`/标注模式：剪贴板图片调用 `pasteExternalImageAsAnnotation()`，按当前视口中心放置为 patch。
- 其他情况：继续调用 `addFiles()`，作为拼接输入图。
- 应用内剪贴板仍优先用于 `Ctrl/Cmd+V`，避免覆盖已有复制流程。

## 新建标注后自动选择

- 非文字对象在 `pointerup` 确认有效后调用 `switchToTransformTool(index)`。
- 点式标号创建后立即切换。
- 文字在编辑框结束提交后切换，避免创建瞬间关闭输入框。
- 水印保持唯一对象，不强制复制多层。

## 兼容

- 工程 JSON 继续使用 annotations 数组，旧项目无需迁移。
- 旧矩形 cutout 无 shape 字段时按 `rect` 处理。
- `index.html` 与 `LongShotStitch.html` 保持完全一致。
