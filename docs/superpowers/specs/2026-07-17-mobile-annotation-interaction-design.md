# LongShotStitch v1.47.0 移动端标注交互统一设计

日期：2026-07-17

## 1. 背景

当前标注系统已具备对象化编辑、缩放、旋转、图层和属性修改，但手机端存在四类体验问题：

1. 文字内容与选择框的垂直位置、尺寸不一致；
2. 单击文字会直接弹出占据大面积屏幕的输入框，妨碍拖动与缩放；
3. 选择对象时属性浮层自动展开并遮住对象；
4. 手机端工具顺序、按钮、输入框和属性布局与桌面端不一致。

## 2. 核心交互模型

### 2.1 选择、变换、属性、编辑分离

- 单击对象：只选中对象，显示移动、缩放、旋转控制点。
- 拖动对象或控制点：直接变换，不自动展开属性。
- 再次轻点已经选中的对象：展开或收起该对象属性。
- 双击文字对象：进入文字内容编辑。
- 新建文字：立即进入贴合对象的行内编辑。
- Esc、单击空白或再次点击选择工具：取消选择并关闭属性。

### 2.2 手机端属性浮层

- 对象被选中时默认关闭，避免遮挡。
- 用户主动打开后，浮层固定在底部工具栏上方。
- 若浮层与选中对象重叠，自动轻微平移画布，使对象保持可见。
- 属性浮层内部紧凑排列，优先单行；超出宽度时局部横向滚动，不整体撑破屏幕。

### 2.3 文字排版

- Canvas 文字、选择框和 textarea 共用同一套字体、行高、内边距和测量结果。
- 文字使用实际字体 ascent/descent 计算基线，使字形在每个行盒中垂直居中。
- 输入框采用 border-box，最终外框尺寸直接等于选择框尺寸。
- 手机端不再使用固定底部 96px 高输入框，改为跟随文字对象的行内编辑框。

## 3. 工具顺序

桌面端和手机端统一为：

1. 选择 / 变换
2. 选区
3. 几何图形
4. 铅笔 / 荧光笔
5. 直线 / 箭头
6. 文字
7. 标号
8. 马赛克
9. 橡皮擦
10. 水印

## 4. 统一控件规范

- 常规紧凑控件高度：28px。
- 图标按钮宽度：28px；危险操作只使用红色语义。
- 数字输入框按实际数字长度控制，不占满整行。
- 文本输入框可伸缩，但不得挤掉同排核心操作。
- 圆角、边框、字体、hover、active、disabled 状态统一。
- 图层与对象操作合并为一行，不保留大面积空白。
- 手机端隐藏不适合触摸操作的 X/Y/旋转数字字段；仍可通过拖动和旋转手柄操作。

## 5. 全工具检查范围

覆盖：选择、选区、形状、画笔、箭头、文字、标号、马赛克、橡皮、水印、粘贴图片和浮动选区。

每种工具检查：

- 创建；
- 选择；
- 移动；
- 缩放或端点调整；
- 旋转（适用时）；
- 属性打开/关闭；
- 属性修改；
- 撤销/重做；
- 删除/复制/图层；
- 手机端边界、遮挡、滚动和触摸操作。

## 6. 验收标准

- 单行文字在选择框内上下视觉居中，输入框与选择框外形一致。
- 手机端单击文字只选中，不自动打开键盘；双击或点击编辑按钮才编辑。
- 手机端选择任意对象不会自动弹出属性遮挡对象。
- 再次轻点同一对象可以稳定打开对应属性。
- 手机和桌面工具顺序完全一致。
- 390px 宽手机视口中，所有工具属性浮层无横向页面溢出。
- 自动测试、JS 语法检查、入口同步检查及真实 Chromium 交互测试通过。
