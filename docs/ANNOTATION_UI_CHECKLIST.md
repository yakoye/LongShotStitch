# LongShotStitch 标注 UI 与交互检查清单

本文件是 `SUPERPOWERS_REQUIREMENTS.md` 的专项执行清单，适用于所有标注、选区、贴图、画布对象和属性面板修改。以后涉及标注功能的版本，在设计、开发、测试和发布前都必须逐项检查。

制定依据：

- 项目现有对象化标注架构；
- Web 工具应紧凑、自然、避免开发者式说明的原则；
- 桌面端与移动端分别设计、分别验证的原则；
- Superpowers 的先设计、先复现、测试驱动、系统调试和完成前验证流程。

## 1. 成熟工具的基本交互模型

### 1.1 选择与编辑分离

- 单击对象：选择对象，显示变换框。
- 拖动对象：移动，不自动弹出遮挡画布的属性。
- 拖动控制点：缩放、旋转或调整端点。
- 手机端再次轻点已选对象：打开或收起属性。
- 桌面端属性常驻左侧面板，不覆盖主画布。
- 双击文字或点击“编辑文字”：修改文字内容。
- 单击空白、再次点击选择工具或按 `Esc`：取消选择。

### 1.2 创建后的状态

- 连续工具：铅笔、荧光笔、自由马赛克、橡皮保持当前工具。
- 单次对象：形状、箭头、文字、标号、矩形马赛克、选区完成后保持新对象选中。
- 连续绘制过程中不显示虚线框；提交后才显示变换控制。
- 创建、选择和属性不得改变用户已经主动选定的子类型，除非产品规则明确要求切到“选择”。

### 1.3 小对象可操作性

- 小文字、短线和小标号的四角控制点不能覆盖整个对象内部。
- 选中对象中心必须保留可靠的拖动命中区。
- 控制点视觉尺寸与命中尺寸可以不同，但不能导致对象无法拖动。
- 放大、缩小、旋转后命中区域必须跟随真实变换。

## 2. 统一组件尺寸

| 组件 | 桌面端 | 手机端 | 规则 |
|---|---:|---:|---|
| 紧凑属性按钮 | 28px 高 | 28px 高 | 图标居中、圆角统一 |
| 底部工具按钮 | 34px 图标按钮 | 至少 44×44px 命中区 | 手机端可横向滚动 |
| 数字输入 | 42–58px 宽 | 42–58px 宽 | 只容纳必要数字 |
| 文本输入 | 88–180px | 最多约 142px | 可伸缩但不挤掉核心操作 |
| 颜色按钮 | 24–28px | 24–28px | 选中态清楚 |
| 属性浮层 | 不超出左侧栏 | 不超出视口、最高约 220px | 内部滚动，不撑破页面 |

统一要求：

- 同类按钮、输入框、颜色块、菜单使用同一高度、圆角、边框和字号。
- 按钮必须有默认、hover、active、disabled、selected 状态。
- 普通操作使用中性色，危险操作使用红色。
- 图标可表达时不增加长文字；必要名称放在 tooltip 或 `aria-label`。
- 属性优先同一行；空间不足时仅让当前属性行内部横向滚动。

## 3. 桌面端检查

- [ ] 顶部工具栏、左侧模式/属性栏、中间画布层级清楚。
- [ ] 标注工具按固定顺序排成整齐网格，不因分隔符产生错位。
- [ ] 对象属性宽度不超过侧栏，不覆盖画布。
- [ ] 坐标、尺寸、角度等精确输入可用。
- [ ] 属性修改后对象仍保持选择，不跳到其他图层对象。
- [ ] 新建文字的指针事件结束后编辑器仍保持焦点。
- [ ] 文字输入框、Canvas 文字和选中框位置一致。
- [ ] 鼠标悬停提示存在，但不形成大段说明。

## 4. 手机端检查

- [ ] 工具顺序与桌面端完全一致。
- [ ] 工具栏按钮至少具有 44×44px 触摸命中区。
- [ ] 工具栏可横向滚动，首尾工具均可到达。
- [ ] 第一次轻点对象只选择和变换，不自动打开属性。
- [ ] 第二次轻点已选对象才打开属性。
- [ ] 属性打开后不遮住选中对象；必要时画布轻微让位。
- [ ] 拖动、缩放或旋转开始时属性自动收起。
- [ ] 文字单击不弹键盘；双击或显式编辑才进入输入。
- [ ] 文字输入框贴合对象，不变成全屏或底部大输入面板。
- [ ] 浮层不越过屏幕左右边界，不被底部浏览器工具栏遮挡。
- [ ] 双指缩放、单指平移和对象拖动不会互相误判。
- [ ] 不依赖 hover；所有隐藏操作都有触摸入口。

每个工具还必须检查：

- [ ] Shift 约束符合该工具语义；
- [ ] 属性面板显示当前对象，而不是当前默认工具；
- [ ] 图层置顶、上移、下移、置底有效；
- [ ] 复制、删除后选择状态正确；
- [ ] 工程保存和重新打开后属性仍可编辑；
- [ ] PNG、JPEG、WebP、SVG 导出结果符合预期。

## 6. 文字专项检查

- [ ] Canvas 编辑期间不重复绘制同一文字，避免重影。
- [ ] 字体、字重、字号、行高、内边距在 Canvas 与 textarea 中一致。
- [ ] 使用实际 ascent/descent 计算基线。
- [ ] 单行文字在框内上下视觉居中。
- [ ] 多行文字每行间距一致，框高随行数变化。
- [ ] 横排和竖排边界正确。
- [ ] 旋转后文字、虚线框和旋转手柄共享中心。
- [ ] 输入框使用 `border-box`，不得靠额外最小宽度撑大而与对象不匹配。
- [ ] 新建文字能持续输入，失焦、提交、取消行为明确。

## 7. 属性浮层专项检查

- [ ] 当前对象 ID 与属性始终一致。
- [ ] 图层重排后属性不跳到相邻对象。
- [ ] 属性输入过程中不整体重建面板导致失焦或只能输入一个字符。
- [ ] 点击外部、收起按钮和 `Esc` 行为明确。
- [ ] 拖动对象时浮层不会截获画布指针。
- [ ] 属性面板内不存在无效控件，例如马赛克的无效颜色选项。
- [ ] 图层、复制、删除、收起在紧凑的一行内，必要时内部滚动。
