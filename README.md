# LongShotStitch

LongShotStitch 是一个网页版本的长截图拼接与裁剪工具。

当前版本：v1.45.0

这个工具的想法参考了 Picsew。换到安卓手机之后，我发现安卓上很难找到一款能媲美 Picsew 的长截图拼接工具，于是仿照它的核心体验做了一个网页版本。它不依赖某个特定手机系统，只要设备有浏览器，就可以在手机、平板、电脑上使用。

## 适用场景

- 多张截图自动拼接成长图
- 对长图进行裁剪、旋转和基础调整
- 添加标注、文字和辅助说明
- 使用矩形、椭圆或自由选区提取画面，并立即移动、缩放或旋转
- 将剪贴板图片粘贴为可再次选中和编辑的独立对象
- 自动判断纵向或横向拼接
- 从视频截图中提取字幕区域并拼成长图
- 将单张图片切成多页、横向/纵向多分或九宫格
- 在不同设备之间保持一致的使用方式
- 不想安装 App，只想打开网页直接处理图片

## v1.45.0 更新

- 修复文字原位编辑重影：编辑期间暂停 Canvas 中同一文字对象的绘制，只保留输入层；提交或取消后恢复最终文字。
- 铅笔、荧光笔和自由马赛克改为连续工具：完成一笔后保持当前工具且不自动显示选择虚线框，切换到“选择 / 变换”后仍可再次编辑。
- 选择模式支持单击空白、再次点击当前“选择”按钮或按 `Esc` 取消选中；拖动画布不会误触取消。
- 橡皮默认改为局部擦除，沿轨迹一点点擦除标注；属性中仍可切换“对象”模式快速删除整个对象。
- 新增扩图：可按上、下、左、右增加像素，或指定最终宽高、比例和九宫格锚点；扩图只扩大画布背景，不拉伸原图和标注。
- 新增“删除并收缩”选区规则：仅允许贴单边、横跨全宽或纵跨全高的矩形选区；普通内部矩形只清除或填充，不进行二维塌缩。
- 内部整宽/整高条带删除后，未跨带标注继续保持对象；跨带标注转换成可再次选择的图片片段，整个操作支持撤销。
- 导出增加指定像素、完整适应、填满裁边和拉伸；支持 128×128、256×256，以及可编辑毫米/DPI 的常用 1 寸、2 寸预设。
- 裁剪比例增加 `25:35` 与 `35:49`，可先框选和调整照片，再按目标像素导出。

完整范围见 `docs/v1.45.0修改记录.md`。

## v1.44.0 更新

- 矩形、椭圆、矩形马赛克和像素选区在创建时支持 `Shift` 等宽高约束；直线和箭头支持 `Shift` 吸附到 45° 方向。
- 铅笔、荧光笔和自由马赛克按住 `Shift` 后，从当前锚点拉出直线；松开后立即恢复自由绘制。
- 选择对象后显示成熟编辑器式控制框：普通对象和自由路径使用四角、四边共 8 个缩放手柄；文字、标号和水印使用四角等比例缩放；直线和箭头直接调整两个端点。
- 四边手柄只调整一个方向，四角手柄同时调整宽高；四角缩放时按住 `Shift` 保持对象原始宽高比。
- 新增图层顺序：置顶、上移一层、下移一层、置底，并支持 `Ctrl/Cmd + ]`、`Ctrl/Cmd + [` 及其 Shift 组合。
- 选区移动或剪切后的原位置可选择透明、白色、黑色或自定义颜色填充，设置随工程、撤销重做和导出保存。
- 文字不再提供背景框选项，旧工程中的文字背景也不再绘制或导出；只在编辑过程中显示辅助轮廓。
- 支持按住 `Alt/Option` 拖动复制对象；画笔、荧光笔和自由马赛克补齐再次选中、移动和 8 点缩放。

完整范围见 `docs/LongShotStitch标注系统任务书.md` 与 `docs/v1.44.0修改记录.md`。

## v1.43.0 更新

- 左侧“工具”模式改名为“标注”，选择入口改为“选择 / 变换”。
- 新增矩形、椭圆和自由选区；选区完成后自动转为浮动图像对象并进入变换状态。
- 浮动选区使用 `patch + cutout` 方式保留透明挖空，移动后的效果接近画图软件的浮动选区。
- 标注模式下粘贴系统剪贴板图片时，图片作为独立对象放入当前画布，可再次选中、移动、缩放、旋转、复制和删除。
- 新建形状、线条、箭头、画笔、标号和马赛克后自动进入选择 / 变换；文字在完成输入后进入选择状态。
- 版本规则调整为 `v1.<功能迭代>.<修复号>`；后续同版本修复依次使用 `v1.43.1`、`v1.43.2`。

完整范围见 `docs/v1.43.0修改记录.md`。

## 在线发布

项目是纯静态网页，可以部署到 Cloudflare Pages、Workers Static Assets、GitHub Pages 或任意静态网站服务。

当前发布流程为：提交并推送 GitHub `main` 分支；Cloudflare 项目连接该仓库后会自动同步、构建并发布。Cloudflare 的部署记录成功后，线上版本才算完成更新。

当前入口文件：

- `index.html`：默认访问入口，直接加载主工具页面
- `LongShotStitch.html`：主工具页面

部署时至少需要上传：

- `index.html`
- `LongShotStitch.html`

如果使用 Cloudflare 的 `Visit` 按钮访问根地址，必须保留 `index.html`，否则根路径 `/` 会找不到页面。

## 本地使用

直接用浏览器打开 `LongShotStitch.html` 即可使用。

也可以打开 `index.html`，它会直接加载同一个工具页面。

## 开发与检查

仓库里包含几个简单检查脚本：

```bash
node tests/entry_check.js
node tests/annotation_maturity_check.js
node tests/annotation_tool_matrix_check.js
node tests/canvas_editing_check.js
node tests/static_check.js
node tests/smoke_check.js
node tests/drag_math_check.js
node tests/tool_interaction_check.js
```

这些脚本用于确认：

- 发布入口 `index.html` 直接加载工具页
- 最新版本页面包含关键功能标记
- 主页面脚本可以被正常解析
- 选择、绘制、删除、像素选区和文字原位编辑规则没有回退

完整测试规则见 `docs/测试规则.md`。
