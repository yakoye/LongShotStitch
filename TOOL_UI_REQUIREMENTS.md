# LongShotStitch 工具属性极简规范

适用版本：v1.9 起，v1.10 补充二次点击编辑与粗细快捷档规则；v1.11 补充导航增强不改变工具区原则。

## 1. 基本原则

- 工具属性区只放可操作控件，不放解释性分类标题。
- 不显示“形状、颜色、线条、类型、样式、粗细、参数、高级”等二级分类文字。
- 工具含义通过图标、按钮状态和悬停提示表达。
- 默认布局紧凑，一行优先；空间不足时自动换行。
- 页面公开界面不写开发说明、教学说明或冗余提示。

## 2. 工具属性展示方式

### 几何图形

直接显示：矩形、圆形、填充开关、颜色块、填充色、线宽、透明度。

### 铅笔 / 荧光笔

直接显示：自由笔、荧光笔、颜色块、线宽。

### 线条 / 箭头

直接显示：直线、单箭头、双箭头、颜色块、线宽。

### 文字

直接显示：文字输入框、字号、颜色块、描边、背景。

### 标号

直接显示：1~10、颜色块、大小。

### 马赛克

直接显示：矩形马赛克、自由马赛克、块大小 / 笔刷大小。

### 橡皮擦

直接显示：橡皮擦大小。橡皮擦只处理标注层，不破坏原图。

### 水印

直接显示：水印文字、字号、颜色、透明度、密度 / 次数。

## 3. 粗细 / 大小调节

所有涉及粗细或大小的控件必须支持三种方式：

1. 图形化预设按钮：用不同宽度的线条直接表示粗细。
2. 鼠标滚轮：鼠标悬停在粗细区域时，向上滚变粗，向下滚变细。
3. 数字设定：保留紧凑数字输入框，可以精确输入数值。

适用范围：矩形边框、圆形边框、自由笔、荧光笔、直线、箭头、双箭头、文字字号、标号大小、马赛克块大小、橡皮擦大小、水印字号。

## 4. 选中对象后的属性

- 选中已有标注对象后，也使用同一套紧凑控件。
- 选中对象的颜色、线宽、字号、透明度等修改应立即作用于对象。
- 不因为选中对象而切换当前工具；当前工具由用户主动选择。

## 5. 桌面端与手机端

- 桌面端工具入口放在左侧“工具”模式下。
- 手机端工具入口保留底部工具栏 / 底部浮层思路。
- 手机端同样不显示二级分类标题，但控件尺寸应适合触摸。


## 6. 粗细快捷档规则（v1.10 起）

- 普通工具的粗细快捷按钮只显示 `1 / 2 / 4 / 8`。
- 普通工具包括：几何图形边框、自由笔、荧光笔、直线、单箭头、双箭头、文字描边、马赛克块大小等。
- `16 / 32` 不作为普通工具的快捷按钮显示，但数字输入框仍允许用户手动输入更大的值。
- 橡皮擦例外：橡皮擦面积工具可以保留 `16 / 32` 快捷档。

## 7. 二次点击编辑规则（v1.10 起）

工具模式点击画布时必须先判断是否点中已有标注对象：

1. 点中已有对象：选中并编辑该对象，不新增对象。
2. 点中空白区域：按当前工具新增对象。
3. 橡皮擦例外：点中已有对象时执行擦除 / 删除。

适用范围：文字、几何图形、铅笔 / 荧光笔、直线 / 箭头、标号、马赛克、水印等工具。

### 文字

- 文字工具点击空白处：新增文字。
- 文字工具点击已有文字：编辑该文字，不新增。

### 水印

- 水印是全局对象，默认只保留一套水印配置。
- 已有水印时，再次点击水印工具或在画布上使用水印，应选中并编辑已有水印，不新增多层水印。

### 其它对象

- 当前为几何、箭头、标号、马赛克、铅笔等工具时，点击已有对象均优先进入编辑。
- 当前工具保持不变；编辑已有对象不应强制切回选择 / 移动。

## v1.12 Mobile Tool Interaction

- Desktop keeps the left-side tool area under the Tools mode.
- Mobile uses a bottom-attached icon dock directly above the bottom mode bar.
- Mobile property panels are not always-on; tapping the active tool icon hides or shows the panel.
- Mobile property panels must remain compact and scroll internally.


## LongShotStitch v1.13 update
- 手机端工具栏必须绑定在底部模式栏上方，不能漂在画布中间遮挡内容。
- 工具属性面板必须提供收起按钮；再次点击当前工具图标也可显示/隐藏属性。
- 工具属性面板在手机端和电脑端都要紧凑，减少 padding、间距和多行堆叠，优先一行内完成。
- 手机端导入图片后默认按屏幕宽度适配，竖向长图优先满宽显示；用户仍可手动放大缩小。
- 页面刷新、关闭、返回导致工程丢失前，必须触发浏览器离开提示。

## 8. v1.14 对象编辑补充

- 手机端工具图标栏必须紧贴底部模式栏上方，不在画布中间悬浮，不留大块空白。
- 选择工具时，只显示工具参数；不显示删除、复制、收起按钮。
- 只有选中已有标注对象时，才显示删除、复制、收起按钮。
- 方框、圆形、马赛克矩形等形状对象再次选中后，必须支持移动、缩放、颜色/填充/线宽修改；非线条对象支持旋转角度设置和旋转手柄。
- 直线、单箭头、双箭头再次选中后，必须支持整体移动、拖动起点、拖动终点，以改变大小、方向和角度。
- 文字对象再次选中后，必须支持移动、字号、颜色、描边、背景、旋转和横排/竖排切换。
- 工具模式不使用 `1 / 2 / 3 / 4 / 5` 作为模式快捷键，避免编辑文字或数字时误触。
- 标号不限制在 1~10；新增标号后自动递增到 11、12、13……
- 选中已有标号后可以编辑数字、移动位置、调整大小和颜色；修改一个标号数字时，后续同批标号按顺序联动递增。

## v1.15 annotation editing rules

- Text annotations must support direct in-canvas editing. Clicking an existing text annotation opens an inline editor at the annotation position; the property panel is not the only place to edit text.
- The inline text editor follows zoom, pan, rotation, and vertical-text mode.
- The selected-object dashed frame and rotation handle must rotate with the object. The frame must not remain axis-aligned while the object is rotated.
- After adding annotations, later clipping/cropping/export should treat annotations as visible image content: annotation rendering is clipped to the visible stitched image/content region rather than floating outside it.

## v1.16 文字画布内编辑规则

- 文字工具在画布内直接编辑时，只保留一个统一的虚线编辑框。
- 画布内文字编辑框不得再使用独立的实色背景、阴影或和选中框错位的第二层边框。
- 新建文字默认进入空白编辑状态，使用 placeholder 提示，不自动选中“输入文字”。
- 文字默认字号收敛为较小尺寸，避免刚创建时遮挡正文内容。
- 文字编辑框的位置、大小、旋转必须与文字对象自己的选区保持一致。


## v1.17 文字与子工具锁定规则

- 文字标注回到稳定的属性面板编辑方式：点击文字后选中、可拖动，文字内容在属性输入框中修改。
- 画布上只保留统一蓝色虚线选中框，不再弹出内联文字输入框，不再出现蓝底“输入文字”。
- 文字默认字号继续保持较小尺寸。
- 工具大类和子类型必须锁定：选中圆形就创建圆形，选中矩形就创建矩形；选中直线、单箭头、双箭头、自由笔、荧光笔、矩形马赛克、自由马赛克等同理。
- 只有用户主动点击其它图标或子类型按钮时，才允许切换当前工具或当前子类型；拖动创建过程中不得自动跳到默认矩形或默认直线。
- 选中已有对象时只显示该对象属性；当前工具和子类型保持不变，后续创建仍使用用户主动选定的工具。


## LongShotStitch v1.18 补充约束

- 文字内容在属性面板输入时必须支持连续输入，不允许因为输入事件重绘属性面板导致只能输入一个字符。
- 默认文字字号应保持较小，适合在截图上做轻量标注。
- 文字旋转时，文字内容、蓝色虚线框、旋转手柄必须使用同一个视觉中心，不能出现文字跑出选择框。
- Backspace / Delete 删除当前选中标注；删除后必须清空选中状态，连续按键不得继续删除其他标注。

## v1.41 工具锁定与像素选区规则

- 所有工具持续锁定，只有用户点击工具或子类型按钮才能切换。
- 方框、圆形、箭头、直线和矩形马赛克必须拖动达到阈值才创建；单击不创建。
- 铅笔、荧光笔、自由马赛克和橡皮必须拖动才生效。
- 箭头和直线选中时只显示端点；自由路径只显示端点提示，不显示虚线包围框。
- 新增像素选取工具，支持复制、透明剪切、粘贴为可移动缩放贴片。
- 标号入口统一使用 `①`，避免与箭头图标混淆。

## v1.42 工具命中与文字编辑规则

本节覆盖第 7 节、v1.16 和 v1.17 中与下列规则冲突的旧要求。

- 只有“选择”工具可以移动、缩放、旋转和编辑已有标注或粘贴图块。
- 绘制工具在已有对象上操作时执行创建，不得移动、缩放或改写已有对象。
- 橡皮只按拖动轨迹删除标注，不进入移动状态。
- 水印工具可以修改唯一的全局水印配置，但不能移动或缩放画布上的已有对象。
- 像素选区粘贴后自动切换到“选择”，最新贴片保持选中并可立即拖动。
- 桌面端新建文字立即打开原位编辑框；选择工具双击已有文字可重新编辑。
- 手机端新建或选择文字后使用独占输入框，以保证输入法能够正常弹出。
- 文字编辑框与 Canvas 使用同一字体、字号、字重、行高、内边距、方向和旋转参数，并随输入实时调整大小。


## v1.43.0 标注、选区与自动变换规则

本节覆盖 v1.41 中“所有工具持续锁定”以及 v1.42 中仅像素选区粘贴后自动切换的旧规则。

- 左侧“工具”模式统一改名为“标注”；桌面端和手机端公开界面不再把该模式显示为“工具”。
- 标注栏第一项为“选择 / 变换”，第二项为“选区”。
- 选区支持矩形、椭圆和自由路径三种形状。
- 有效选区完成后立即提取为可编辑 `patch`，同时用同形状 `cutout` 挖空源位置，并自动进入“选择 / 变换”。
- 选区生成的 patch 支持移动、缩放、旋转、透明度、复制、删除、撤销、重做和工程保存。
- 标注模式且已有底图时，系统剪贴板图片粘贴为 patch；其它模式或空工程时仍作为拼接输入图。
- 所有 patch 在取消选择后仍可通过“选择 / 变换”再次选中和编辑。
- 形状、线条、箭头、自由绘制、标号和马赛克完成创建后自动回到“选择 / 变换”，新对象保持选中。
- 新建文字在编辑提交后进入“选择 / 变换”；文字输入过程中不得提前关闭编辑框。
- 绘制工具在已有对象上仍执行创建，不直接移动底层对象；再次编辑已有对象必须使用“选择 / 变换”。
- 水印继续保持单一全局对象规则。


## v1.44.0 成熟标注交互规则

- 普通对象和自由路径使用四角、四边共 8 个缩放点；边点单轴、角点双轴。
- 角点配合 Shift 保持原始比例；文字、标号、水印只显示角点并始终等比例。
- 直线和箭头只保留两个端点；端点或新建线条配合 Shift 吸附 45°。
- 铅笔、荧光笔、自由马赛克按住 Shift 时从当前锚点拉直。
- 选中对象配合 Shift 拖动锁定水平/垂直；Alt/Option 拖动复制。
- 每个对象属性浮层提供置顶、上移、下移、置底，不新增大段说明文字。
- 选区属性提供透明、白、黑、自定义颜色的原位置填充。
- 文字不提供背景框，不导出方框；编辑辅助轮廓只在编辑状态显示。
- 手机端标注横向工具栏保留常用工具并可滚动，必须包含水印入口。


## v1.45.0 画布编辑与输出规则

本节覆盖前文中与连续工具、橡皮、文字背景和选区收缩冲突的旧要求。

- 文字原位编辑时必须暂停绘制同一 Canvas 文字，避免输入层和最终对象重影。
- 铅笔、荧光笔、自由马赛克是连续工具；完成一笔后保持当前工具，不显示选择框。
- 选择工具单击空白、重复点击选择按钮或按 `Esc` 可取消选中；空白拖动仍平移画布。
- 橡皮默认局部擦除，另提供对象删除模式；二者只作用于标注层。
- 文字不提供背景框，不绘制旧工程中的文字背景字段。
- 扩图通过增加四边画布和背景实现，不拉伸任何已有对象。
- “删除并收缩”仅允许矩形单边贴边、横跨全宽或纵跨全高；普通内部矩形只清除/填充。
- 导出支持指定目标像素和毫米/DPI 换算，缩放只作用于最终文件。

## v1.46.0 连续绘制、对象属性与导出面板规则

- 铅笔、荧光笔和自由马赛克按住拖动期间只显示实际轨迹，不显示选区虚线框、缩放点、首尾点或旋转点。
- 指针松开并形成有效轨迹后，轨迹立即保持选中；显示首尾标记、8 点缩放框和旋转控制，但当前连续绘制工具保持激活，下一次按下仍创建新轨迹。
- 所有标注对象必须通过统一的稳定选择入口选中。选择记录同时保存对象 ID 和当前索引，图层重排、复制、粘贴或数组变化后优先按 ID 恢复。
- 单击任意对象必须稳定显示该对象自己的属性，不得偶发隐藏、显示工具默认属性或显示上一个对象的属性。
- 导出菜单桌面端最大宽度为 360 px，移动端不超过视口；尺寸输入框只保留足够容纳常见数字的宽度。
- 输出宽高使用约 76 px 输入框，毫米尺寸使用约 72 px，DPI 使用约 82 px；不得因 `input[type=number]` 的固有宽度把浮层撑开。

## v1.47.0 移动标注、文字与统一控件规则

本节覆盖 v1.42 中“手机端选择文字立即进入独占输入框”以及 v1.46 中“手机端单击对象立即显示属性”的冲突规则。

- 手机端单击对象只选择并显示变换控制，不自动打开属性，不自动进入文字内容编辑。
- 再次轻点已经选中的对象才展开或收起对应属性；双击文字或点击显式编辑按钮才修改文字。
- 拖动、缩放、旋转和端点调整开始时必须收起手机属性，避免浮层截获指针或遮住对象。
- 用户主动打开属性后，如果浮层与对象重叠，应让画布轻微避让，使对象仍然可见。
- 桌面端属性保留即时显示，但必须完全限制在左侧栏内，不得覆盖主画布。
- Canvas 文字、虚线框和 textarea 共用实际字体 ascent/descent、行高、内边距和 `border-box` 尺寸。
- 手机文字编辑器不得设置与真实对象无关的最小宽度；新建文字和已有文字编辑框必须贴合对象。
- 小对象中心必须保留拖动命中区，四角触摸控制点不能覆盖整个对象。
- 手机端工具顺序与桌面端完全一致：选择、选区、形状、画笔、箭头、文字、标号、马赛克、橡皮、水印。
- 桌面端十个标注工具使用稳定 5×2 网格；不得用额外分隔元素打乱网格顺序。
- 常规属性控件统一 28px 高；数字输入框按数字长度收窄；文字输入框、颜色块、按钮圆角和边框统一。
- 图层、复制、删除和收起保持紧凑单行；空间不足时该行内部横向滚动，不让整个页面或侧栏溢出。
- 全部专项规则和逐工具验收矩阵见 `docs/ANNOTATION_UI_CHECKLIST.md`。
